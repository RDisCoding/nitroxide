What you’re seeing is expected.

In the current implementation, both modes are still operating over very structured, low-entropy data, so they often converge on similar relations. The LLM is not getting some magical extra source of truth; it is mostly seeing node metadata like name, role, entity, and maybe a few direct fields. SpaCy is also not doing free-form reasoning from nowhere, because it is still being fed structured summaries or templated text. That’s why the outputs look close.

The key point is this: descriptions only help if they add information that is not already obvious from the node fields. If a description is just a paraphrase of the metadata, it will not change much. For example, “Rahul Sharma is a Principal Engineer on Core Platform” is basically the same signal as name + role + team. The model has little new to learn from that.

So for your case:
- If you give only raw node metadata, the LLM can still infer some relations, but only weakly and inconsistently.
- If you give descriptions that contain genuinely new facts, the LLM gets better.
- If you give relation-oriented text like “Rahul collaborates with Vikram on Sync Engine,” that is much better than a generic node description.

That also explains the difference between the comparison outputs:
- The spaCy run is mostly extracting from sentence patterns tied to graph structure.
- The LLM run is reading the same kind of compact structure and making similar guesses.
- The extra descriptions did not help much because they were derived from the graph itself, not independent evidence.

So the practical conclusion is:
- Descriptions are optional, not essential.
- They only matter if they contain real extra semantics.
- For best results, provide relation text or notes, not just node summaries.
